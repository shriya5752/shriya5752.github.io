§ 1 — Traditional Programming vs Machine Learning
Traditional Programming Data + Rules → Output You write the if/else. The computer executes exactly what you tell it.
Machine Learning Data + Output → Rules You show examples. The computer finds the rules itself.
Deep Learning Raw Data + Output → Complex Rules Learns layered representations automatically, end-to-end.
The Three Things Every ML System Needs
📊 Data Examples of inputs and (usually) outputs. The raw material. Garbage in, garbage out.
🧮 A Model A mathematical function that maps inputs to outputs. Its internal parameters are what get learned.
📉 A Loss Function A way to measure how wrong the model is right now — so it knows which direction to improve.

§ 5 — 9 Concepts Every ML Person Must Know
Concept 01

Overfitting vs Underfitting

Overfitting: memorises training data, fails on new data. Underfitting: too simple to capture the pattern. The sweet spot between them is the whole goal of training.

Concept 02

Bias-Variance Tradeoff

High bias = too simple, wrong systematically. High variance = too complex, right on training only. Every modelling decision is a negotiation between these two.

Concept 03

Gradient Descent

Follow the slope of the loss function downhill, step by step, until you reach the minimum. The learning rate controls how big each step is.

Concept 04

Regularisation

Penalise complexity to prevent overfitting. L1/L2 add penalty terms. Dropout randomly kills neurons. BatchNorm stabilises activations.

Concept 05

Train / Val / Test Split

Train: model learns. Validation: tune hyperparameters. Test: touch exactly once, at the very end. Never let test data influence any training decisions.

Concept 06

Embeddings

Turning discrete things (words, images, users) into dense vectors where similar things are close together in space. Foundational to all of modern AI.

Concept 07

Attention Mechanism

Lets a model selectively focus on different parts of the input when producing each output. The core insight behind transformers and all LLMs.

Concept 08

Transfer Learning

Take a model trained on one task, fine-tune it on another. Why we can train GPT on all text then adapt it to any task. The dominant paradigm in modern ML.

Concept 09

The Scaling Laws

More data + more compute + bigger models = predictably better performance. This empirical observation drove the entire LLM revolution after 2020.

§ 6 — The History in 12 Moments
1950

Turing's Question

"Can machines think?" Alan Turing proposes the Turing Test. The field is conceptually born.

1957

The Perceptron

Rosenblatt builds the first trainable neural network. Single layer, linear — but it learns from data.

1986

Backpropagation

Rumelhart, Hinton & Williams. The algorithm that makes multi-layer networks learnable. Foundation of all deep learning.

1997

LSTMs

Solve the vanishing gradient problem for sequences. Language modelling becomes viable for the first time.

1998

LeNet & CNNs

LeCun trains CNNs to read handwritten digits. Computer vision as a learned discipline begins.

2001

Random Forests

Breiman's ensemble method. Still the hardest to beat on tabular data today, 25 years later.

2012

The AlexNet Moment

A deep CNN crushes ImageNet by a massive margin. GPU training becomes standard. The DL era begins.

2014

GANs

Goodfellow proposes adversarial training. Two networks in competition. Generative AI is born.

2017

Attention Is All You Need

The Transformer paper. Replaces RNNs entirely. Everything that comes after builds on this.

2018–20

BERT & GPT-3

Pre-training + fine-tuning becomes the paradigm. Scale begins to matter above all else.

2022

ChatGPT

RLHF aligns GPT to follow instructions. 100M users in 2 months. ML enters the mainstream.

2024–25

The Reasoning Era

o1, DeepSeek R1. Chain-of-thought at scale. Models learn to think before answering. The frontier shifts again.

§ 7 — Mental Models for Thinking About ML
⚖️

ML is always an optimisation problem

Every algorithm — from linear regression to GPT — is minimising some loss function. Understand what's being optimised and why, and you understand the algorithm. Full stop.

🗜️

Learning = compression

A model learns by finding the smallest description of the patterns in data. Good models compress efficiently. Overfitted models just memorise — no real compression, no real learning.

🗺️

Features are the map, not the territory

What you feed the model matters as much as the model itself. Feature engineering in classical ML, representation learning in deep learning — both are about building better maps of the data space.

📈

Scale changed everything

Before ~2020, algorithmic cleverness mattered most. After, compute + data often wins. Understanding both regimes makes you a complete ML practitioner.
