Component Lifecycle:#

Mounting constructor() ‚Üí Initialize state ‚Üì render() ‚Üí Show on screen ‚Üì componentDidMount() ‚Üí Component is ready! (fetch data, start timers)

Updating [User interacts, state/props change] ‚Üì shouldComponentUpdate() ‚Üí Should I update? ‚Üì (if true) render() ‚Üí Update screen ‚Üì componentDidUpdate() ‚Üí Update complete!

Unmounting [Component no longer needed] ‚Üì componentWillUnmount() ‚Üí Cleanup and goodbye!

Methods:#

Mounting

methods:

-constructor()

-render()

-componentDidMount(): initiate API calls in order to fetch data

eg: fetching data

  1. constructor() ‚Üí Set isLoading: true
  2. render() ‚Üí Shows “Loading…”
  3. componentDidMount() ‚Üí Fetch API
  4. API returns data ‚Üí setState()
  5. render() ‚Üí Shows actual user data

Updating

-getDerivedStateFromProps():

‚Äîinvoked right before rendering

‚Äîcompare incoming props with current props

‚Äîmake state updates or logic adjustments

-render()

-shouldComponentUpdate(nextProps,nextState):

‚Äîcalled before every re-render

‚Äîdecides whether a component should re-render or not by comparing current state and nextstate

‚Äîperformance optimization

‚Äîreturns true/false

-GetSnapshotBeforeUpdate()

-ComponentWillUpdate():

-ComponentDidUpdate():

‚Äîinvoked immediately after update occurs

‚Äîmake API calls based on props changes, update 3rd party library as data changes

  • Runs immediately AFTER an update occurs (state or props changed)
  • Does NOT run on initial render (only on updates)
  • Receives previous props and state as parameters

Unmounting

-componentWillUnmount(): cancel timers,cancel api requests

  • Runs right BEFORE the component is removed from the DOM
  • Runs only ONCE when component is destroyed
  • Last chance to clean up

React Router:#

-lets you create a Single Page Application (SPA) with multiple “pages” without actually reloading the browser

-Without Router:

Clicking a link reloads the entire page from the server

With Router:

Clicking a link just swaps out components instantly, no reload!

  1. In modern React, how are properties transferred from a parent component to a child component in React.js and what is the benefit of using the spread operator for this purpose? Explain using a simple example

-In modern React, properties (props) are transferred from a parent component to a child component by passing them as attributes in JSX. The child component receives these props as a parameter (either as an object or through destructuring)

// PARENT COMPONENT function Parent() { return ( ); }

// CHILD COMPONENT function Child(props) { return ( Name: {props.name} Age: {props.age} City: {props.city} ); }

The spread operator (...) allows you to pass multiple props efficiently without writing each one individually.

  1. What is the purpose of keys in React? If the key attribute is absent while using the map function, React throws a KeyNotFoundError. True or False. In either case explain why.(1+1+2)

-Keys help React identify which items in a list have changed, been added, or been removed. They give each element a stable identity, allowing React to efficiently update the DOM by tracking elements across re-renders instead of recreating the entire list.

-React does NOT throw a KeyNotFoundError when the key attribute is missing. -When you use map() without keys, React shows a WARNING in the browser console, not an error:
